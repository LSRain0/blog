## 费曼学习法

理查德·费曼说过："如果你不能用简单的语言解释一个概念，说明你还没有真正理解它。"

这就是费曼学习法的核心：

1. **选择一个概念**
2. **用简单的语言解释它**（就像教给一个外行）
3. **发现解释不清楚的地方**——那就是你还没理解的地方
4. **回去学习，再次尝试解释**

写作本质上就是这个过程的文字版本。

## 写作的几个好处

### 整理思路

学习时，知识在脑海中是零散的。写作迫使你把这些零散的点连成线，再织成网。

比如学习电磁学时，库仑定律、电场、电势、高斯定理——这些都是独立的知识点。但当你试图写一篇完整的笔记时，你会发现它们之间的逻辑关系：

> 库仑定律 → 电场定义 → 高斯定理 → 电势 → 能量

这种结构化的理解，比零散记忆牢固得多。

### 暴露盲区

不写的时候，你觉得自己懂了。一写，才发现有好多地方说不清楚。

这其实是好事。发现盲区是进步的第一步。

### 长期记忆

写下来的东西不会忘。即使忘了，回头看一遍就能恢复。

而不写下来的东西，过几个月可能就真的忘了。

### 帮助他人

你的笔记可能对别人也有用。一个和你水平相近的人，看你写的笔记，可能比看教科书更容易理解。

## 为什么选择博客

有人用纸质笔记本，有人用 Notion，有人用 Obsidian。我选择博客，原因是：

- **公开性**：公开写作会促使你更认真
- **Markdown**：支持公式和代码，适合理工科
- **版本控制**：Git 记录每一次修改
- **分享便利**：发个链接就行

## 关于「完美主义」

很多人不开始写，是因为觉得写不好。

但第一版永远不需要完美。先写出来，再慢慢改。这篇文章也是这样——先有框架，再填充细节。

```python
# 学习的过程
knowledge = []
while learning:
    new_thing = learn()
    knowledge.append(new_thing)
    write_down(knowledge)  # 写作巩固理解
    review_and_update()    # 回顾更新
```

## 结语

如果你也在学习，不妨试试写点什么。不需要发表，写给自己看也行。

重要的是：**把脑子里模糊的东西，变成纸上清晰的文字。**
